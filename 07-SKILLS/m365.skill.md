# Microsoft 365 DevOps Skills

Key concepts for M365 automation and governance.

## Microsoft 365 Tenants

- **Tenant ID**: UUID identifying your entire M365 org
  - Find: Azure AD > Properties > Tenant ID
  - Format: `a1b2c3d4-e5f6-47a8-9c1d-2e3f4a5b6c7d`
- **Domains**: Primary domain (*.onmicrosoft.com) + custom domains
- **Multi-tenant**: Some orgs manage multiple tenants (compliance separation)
- **Subscriptions**: M365 Business, Enterprise, Education SKUs

## Microsoft Graph API

- **Endpoint**: `https://graph.microsoft.com/v1.0` (v1.0 = stable, beta = preview)
- **Authentication**: OAuth2 (interactive) or Client Credentials (service principal)
- **Permissions**: 
  - Delegated (on behalf of user)
  - Application (without user context, service principal only)
- **Scopes**: e.g., `Mail.Read`, `User.Read.All`, `Group.Read.All`
- **Rate limiting**: 15 requests/second per app/tenant (handle 429 responses)
- **Pagination**: Large result sets use `@odata.nextLink`; loop until exhausted

## Azure AD / Entra ID

- **Users**: Directory entries with UPN (user@tenant.onmicrosoft.com)
- **Groups**: Security groups (dynamic or static), M365 groups
- **Applications**: App registrations for script authentication
- **Service principals**: Represent app in tenant; holds credentials
- **Multi-factor auth**: Enforce via Conditional Access policies
- **Hybrid identity**: Sync on-premises AD with cloud (Azure AD Connect)

## Conditional Access

- **Purpose**: Grant/deny access based on risk signals (device, location, app)
- **Policies**: Evaluate in order; first match applies
- **Conditions**: User/group, cloud app, device, location, client app
- **Controls**: Require MFA, compliant device, approved app, deny access
- **Best practice**: Exclude emergency access account; test in report-only mode first
- **Common rules**:
  - Require MFA for risky logins
  - Block legacy auth
  - Require managed device for data access

## Secure Score

- **Metric**: Percentage (0–100) of completed security recommendations
- **Calculation**: Actions completed / Total available actions
- **Benchmark**: Compare to similar orgs; set improvement targets
- **Review frequency**: Monthly assessment; quarterly improvement planning
- **Common low-hanging fruit**:
  - Enable MFA for all users
  - Require strong passwords
  - Enable password spray protection
  - Block legacy authentication

## User & Group Management

- **Create user**: Email format `firstname.lastname@tenant.onmicrosoft.com`
- **Bulk operations**: Use PowerShell + Graph API for large changes
- **Group nesting**: Groups can contain other groups
- **Dynamic groups**: Membership rules (e.g., department = "Engineering" OR manager changed)
- **Lifecycle**: Onboarding (provision), offboarding (disable, archive mailbox)

## Exchange Online (Email)

- **Mailbox**: User's email store (1 per user)
- **Distribution lists**: Email group (no mailbox)
- **Shared mailbox**: Shared across team (no license cost)
- **Transport rules**: Auto-apply labels, encrypt, redirect based on conditions
- **Data loss prevention (DLP)**: Prevent exfiltration of sensitive data
- **Retention policies**: Auto-delete or archive old email

## SharePoint Online

- **Sites**: Team sites, communication sites, subsites
- **Lists**: Structured data (similar to Excel sheets)
- **Libraries**: Document storage with versioning
- **Permissions**: Inherited from site; can override per item
- **Search**: Full-text search across org (respects permissions)
- **Retention**: Document retention policies (e.g., delete after 7 years)

## Teams & Collaboration

- **Teams**: Chat, channels, files, meetings, apps
- **Channels**: Organizing space within team (public or private)
- **Guest access**: External users (B2B); separate permissions
- **Compliance**: Recording, transcription, message retention settingsConfiguration**: Retention, guest access, app availability

## Compliance & Privacy

- **Data subject access requests (DSAR)**: User right to export personal data
- **GDPR**: EU regulation (right to be forgotten, consent tracking)
- **eDiscovery**: Search and hold email/docs for legal litigation
- **Information barriers**: Prevent communication between groups (compliance walls)
- **Audit logs**: Track who accessed/changed what data (retention policy)

## APIs & Cmdlets

- **PowerShell**: ExchangeOnlineManagement, MSGraphAPI modules
- **CLI**: `mgc` (Microsoft Graph CLI) for interactive queries
- **SDK**: `msgraph-sdk-python`, `@microsoft/msgraph-sdk-javascript`
- **REST**: Direct HTTP calls with Authorization header
